# Version 4 — Application Platform on EKS

## Goal

Refactor into a three-layer application platform. Move platform capabilities from ArgoCD to Terraform. Keep ArgoCD focused on apps.

---

## Layers

| Layer        | Role                                              | Tooling                                             | IaC              |
| ------------ | ------------------------------------------------- | --------------------------------------------------- | ---------------- |
| **Infra**    | AWS foundation (VPC, EKS control plane, IAM)      | AWS, EKS                                            | Terraform        |
| **Platform** | Cluster capabilities (compute, network, sec, obs) | Karpenter, Istio, ALBC, ESO, external-dns, cert-mgr | Terraform+Argocd |
| **App**      | Individual workloads                              | ArgoCD, Argo Rollouts                               | Helm / manifests |

- Terraform: manage infra; boostrap step like argocd install; secret creation;
- Argocd: Manage capabilities

---

## Capabilities

### Compute — Karpenter

| Node class | Instance family | Purpose            |
| ---------- | --------------- | ------------------ |
| general    | m6i / m7i       | default workloads  |
| database   | r6i / i4i       | high I/O, stateful |
| gpu        | g5 / g6         | ML / AI            |

### Storage — EBS CSI

| Class      | Type | Reclaim | Use            |
| ---------- | ---- | ------- | -------------- |
| `gp3`      | gp3  | Delete  | default        |
| `gp3-iops` | gp3  | Retain  | databases, WAL |

### Network

| Concern       | Tooling                |
| ------------- | ---------------------- |
| CNI + NetPol  | VPC CNI (NP enabled)   |
| Gateway API   | Istio ambient          |
| Load Balancer | AWS Load Balancer Ctrl |
| DNS           | external-dns           |

### Security

| Concern        | Tooling       |
| -------------- | ------------- |
| Sensitive data | ESO           |
| mTLS           | Istio ambient |
| Network policy | VPC CNI       |
| TLS certs      | cert-manager  |
| Policy         | Kyverno       |

### Observability

| Signal    | Tooling            |
| --------- | ------------------ |
| Metrics   | Prometheus + Alloy |
| Logs      | Loki (S3 backend)  |
| Dashboard | Grafana            |

### GitOps

| Concern           | Tooling       |
| ----------------- | ------------- |
| Continuous deploy | ArgoCD        |
| Release strategy  | Argo Rollouts |

---

## State Layout

Split Terraform state per layer to limit blast radius:

- `infra.tfstate` — VPC, EKS control plane, IAM
- `platform.tfstate` — Helm releases, storage classes, CRDs
- ArgoCD state lives in Git (app manifests)
